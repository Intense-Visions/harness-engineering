## As-Built Source Facts (authoritative — document these, NOT the spec's original assumptions)

- **complexity-trend-up-30d** — `.harness/arch/timeline.json`; reads
  `snapshots[].metrics.complexity.value` keyed by `capturedAt`. Single snapshot exists today,
  so the trend is cosmetic until arch snapshots accrue. Source const: `arch/timeline.json`.
  threshold warn 5 / alert 15, betterDirection **down**, unit `count`.
- **baseline-auto-update-count** — `git log --since=30.days -- '*-baselines.json'`, counting
  commits authored by `github-actions[bot]` whose subject begins `chore: refresh baselines`
  (both conditions required; the glob covers arch/coverage/benchmark baselines). threshold
  warn 1 / alert 5, betterDirection **down**, unit `count`.
- **coverage-trend-down-30d** — derived from **git history of `coverage-baselines.json`**
  (mean of every package's `lines` % per CI commit). No native coverage time-series exists.
  Degrades to `error` with an enable hint when no history is found. threshold warn -1 /
  alert -5, betterDirection **up**, unit `%`.
- **pr-merged-without-multi-persona-review** — `gh pr list` (merged, 30d) + per-PR review
  bodies via the gh reviews API; a PR is "reviewed" iff a review body contains the
  `## Assessment:` marker emitted by the multi-persona review pipeline. Depends on `gh`;
  degrades to `error` when gh is unavailable. threshold warn 1 / alert 3, betterDirection
  **down**, unit `count`.
- **eval-fail-rate** — knowledge-graph `execution_outcome` nodes; reads `metadata.result`
  (`'success' | 'failure'`) and `metadata.timestamp` (ISO). Returns `pending`/value null when
  zero such nodes exist (until `harness:outcome-eval` publishes them). threshold warn 5 /
  alert 10, betterDirection **down**, unit `%`.
- **Mechanism** — time-series signals derive their 30d window on first compute, persist daily
  points to `.harness/signals/timeline.json` (version 1; `{ signals: { <id>: SignalPoint[] } }`),
  and read cache thereafter; corrupt/missing cache soft-fails to re-derivation.
- **Landing** — `/` redirects (Navigate replace) to `/s/signals`; chat remains reachable at
  `/t/:threadId`.

**Depends on:** none | **Files:** docs/knowledge/decisions/0037-graph-as-cross-skill-signal-contract.md | **Category:** integration

2. Body sections (match 0036/0007 style; REFERENCE the spec's Decisions #2, do not restate it):
   - `## Context` — `eval-fail-rate` must consume `harness:outcome-eval`'s output, but the two
     are built in parallel. `execution_outcome` is already a registered graph node type
     (`packages/graph/src/types.ts`); the dashboard already reads the graph via `GraphStore`.
     Cite Spec 534 Decision #2 (`docs/changes/five-signal-dashboard-panel/proposal.md`) as the
     originating decision.
   - `## Decision` — the **knowledge graph node shape is the only shared contract** between the
     dashboard signal and the outcome-eval producer. The dashboard reads
     `findNodes({ type: 'execution_outcome' })` and depends only on `metadata.result`
     (`'success'|'failure'`) and `metadata.timestamp` (ISO). Zero import/file coupling to
     outcome-eval. Absence of nodes → `pending`, not error. Aligns with STRATEGY.md's
     graph-as-durable-substrate bet.
   - `## Consequences` — Positive (parallel development, reversible, future signals/skills reuse
     the pattern); Negative (contract is implicit in the node shape — a producer changing
     `metadata.result`/`timestamp` silently breaks the consumer; the only guard is this ADR +
     the provider's defensive skipping); Neutral (consumer soft-fails on missing/unloadable
     graph).
   - `## Alternatives considered` — (a) direct import of outcome-eval types: rejected, couples
     parallel work; (b) a shared TS interface package: rejected as premature for one consumer;
     (c) a file-based handoff under `.harness/`: rejected, the graph is already the durable
     substrate.
   - `## Implementation` — `packages/dashboard/src/server/signals/providers/eval-fail-rate.ts`
     (reads the nodes, pending path); `packages/graph/src/types.ts` (`execution_outcome` node
     type). Note the marker fields it depends on.
   - **Constraint:** no `#NNN` hex; write "Spec 534" not "#534".
